# Ghost in the System: IT Audit Investigation Game

โครงการนี้เป็นส่วนหนึ่งของรายวิชา ICT25367 จัดทำขึ้นในรูปแบบสื่อการเรียนรู้เชิงปฏิสัมพันธ์ประเภทเกมสืบสวนดิจิทัล (Investigation Game) ที่ผสมผสานรูปแบบ Visual Novel บน Web Browser

## 📌 หลักการและเหตุผล (Why we built this)
แม้การเรียนการสอนในชั้นเรียนจะช่วยเสริมสร้างองค์ความรู้ด้านการตรวจสอบเทคโนโลยีสารสนเทศ (IT Audit) ได้เป็นอย่างดี แต่การนำความรู้ดังกล่าวไปประยุกต์ใช้ในสถานการณ์จริงนั้น จำเป็นต้องอาศัยการบูรณาการร่วมกับทักษะอื่นๆ เพื่อให้เกิดความเชี่ยวชาญ โครงการนี้จึงถูกสร้างขึ้นเพื่อสะท้อนผลสัมฤทธิ์ทางการเรียนรู้ผ่านการบูรณาการองค์ความรู้ทางเทคนิคออกมาอย่างเป็นรูปธรรม 

ผู้เล่นจะได้รับบทเป็น Junior IT Auditor ที่ต้องเข้าไปสืบสวนเหตุการณ์ข้อมูลรั่วไหล (Data Breach) ในบริษัทเอกชน ผ่านการอ่านบทสนทนา เก็บหลักฐาน และการตัดสินใจเชิงจริยธรรม ซึ่งโปรเจกต์นี้ได้นำความรู้จากรายวิชาหลักมาทำงานร่วมกับทักษะการออกแบบประสบการณ์ผู้ใช้ (UX Design) และการเขียนโปรแกรม (HTML, CSS และ JavaScript) เพื่อสร้างโครงสร้างเนื้อเรื่องแบบมีเงื่อนไขอย่างเป็นระบบ

## 🎯 วัตถุประสงค์ของโครงการ (Objectives)
* เพื่อสร้างสื่อการเรียนรู้เชิงปฏิสัมพันธ์ที่ช่วยส่งเสริมความเข้าใจในกระบวนการตรวจสอบการควบคุมการเข้าถึงระบบ (Logical Access Control) อย่างเป็นขั้นตอน
* เพื่อเปิดโอกาสให้ผู้เรียนได้ฝึกทักษะการคิดวิเคราะห์ข้อมูล Log อ่านหลักฐาน และการตัดสินใจเชิงจริยธรรมในสถานการณ์จำลองที่ใกล้เคียงกับการปฏิบัติงานจริง
* เพื่อบูรณาการความรู้ด้าน IT Audit จริยธรรมวิชาชีพ และทักษะการพัฒนา Web Application เข้าด้วยกันอย่างเป็นรูปธรรม
* เพื่อนำเสนอผลสัมฤทธิ์ทางการเรียนในรูปแบบที่สร้างสรรค์ สามารถวัดผลได้จริง และสำเร็จลุล่วงได้ภายในระยะเวลาที่กำหนด

## 📚 ทฤษฎีและหลักการที่ใช้อ้างอิง (Theoretical Framework)
เนื้อหา กลไกการตัดสินใจ และระบบประเมินผลภายในเกม ถูกออกแบบโดยอ้างอิงจากหลักวิชาการและมาตรฐานการตรวจสอบปฏิบัติงานจริง ดังนี้:

### 1. หลักการและจรรยาบรรณของนักตรวจสอบ (Audit Principles - อ้างอิงตามมาตรฐาน ISO 19011)
ตัวเกมได้จำลองสถานการณ์เพื่อทดสอบผู้เล่นตามแนวคิด **"7 เสาหลักแห่งความเชื่อมั่น (The 7 Pillars of Trust)"** โดยมีแกนหลักที่ส่งผลต่อคะแนนความซื่อสัตย์ (Ethics Meter) ในเกม ได้แก่:
* **Independence (ความเป็นอิสระ):** ผู้เล่นต้องรักษาความเป็นกลาง ไม่โอนอ่อนตามคำขอร้องหรือความสัมพันธ์ส่วนตัวกับตัวละครผู้ถูกตรวจสอบ
* **Evidence-based Approach (หลักฐานเป็นหลัก):** ผู้เล่นไม่สามารถกล่าวหาบุคคลอื่นลอยๆ ได้ แต่ต้องอาศัยการรวบรวมเบาะแสจาก Log และอีเมล เพื่อใช้ยื่นผ่าน "กระดานหลักฐาน" ยืนยันข้อเท็จจริง
* **Integrity & Fair Presentation (ความซื่อสัตย์และการนำเสนออย่างเป็นธรรม):** สะท้อนผ่านตัวเลือกในช่วงท้ายเกม ที่ผู้เล่นต้องเลือกรายงาน "ข้อเท็จจริง" อย่างตรงไปตรงมา แทนการใช้อารมณ์ใส่ความบุคคล

### 2. ข้อสรุปจากการตรวจสอบ (Audit Conclusion)
ระบบการให้คะแนนและฉากจบของเกม (Final Evaluation) จะประเมินจากความสามารถของผู้เล่นในการจัดทำรายงานและสรุปภาพรวมของข้อบกพร่อง ซึ่งอ้างอิงจากการจัดประเภทข้อสรุป 4 ระดับ ได้แก่:
* **มีประสิทธิผล (Effective - Green):** ระบบควบคุมได้มาตรฐาน ไม่พบความเสี่ยง (สถานการณ์อุดมคติ)
* **มีเงื่อนไข (Qualified - Yellow):** โดยรวมมีประสิทธิภาพ แต่พบข้อบกพร่องบางประการที่ต้องแก้ไข
* **ไม่มีประสิทธิผล (Ineffective - Red):** พบข้อบกพร่องร้ายแรง เช่น บัญชีผู้ใช้ไม่ถูกระงับ (Terminated User) จนนำไปสู่การรั่วไหลของข้อมูล ซึ่งเป็นหัวใจหลักของคดีในเกมนี้
* **ไม่แสดงความเห็น (Disclaimer of Opinion - Grey):** เกิดขึ้นในกรณีที่ผู้เล่นรวบรวมหลักฐานไม่เพียงพอจนไม่สามารถสรุปรูปคดีได้
